
## Conventions (apply as project grows)

- Keep source code under a `src/` directory
- Place tests alongside source or in a top-level `tests/` directory
- Store configuration files at the root
- Do not commit secrets or `.env` files — use `.env.example` as a template
- Use clear, descriptive names for files and folders (lowercase, hyphens for spaces)
